## Математические алгоритмы

### 1. Линейная интерполяция

Используется для приближённого вычисления значения функции между двумя известными точками.

Формула:

$$t = \frac{x - x_1}{x_2 - x_1}$$

$$y = y_1 + t (y_2 - y_1)$$

### 2. Интерполяция полиномом Ньютона

Используется полином в форме Ньютона:

$$P(x)=a_0 + a_1(x-x_0) + a_2(x-x_0)(x-x_1) + \ldots$$

Коэффициенты вычисляются через таблицу разделённых разностей:

$$f[x_i] = y_i$$

$$f[x_i, x_{i+1}] = \frac{f[x_{i+1}] - f[x_i]}{x_{i+1}-x_i}$$

$$f[x_i, x_{i+1}, x_{i+2}] = \frac{f[x_{i+1},x_{i+2}] - f[x_i,x_{i+1}]}{x_{i+2} - x_i}$$

Последовательность первых элементов каждой строки таблицы разностей формирует коэффициенты `[a_0, a_1, a_2, …]`.

Вычисление значения полинома выполняется по схеме Горнера для формы Ньютона:

$$P(x) = a_0 + (x - x_0)\big[a_1 + (x - x_1)[a_2 + \cdots]\big]$$

## Ключевые элементы реализации

### Модуль `interpolation.clj`

#### Полиморфная архитектура

Используются мультиметоды для создания единого интерфейса для всех алгоритмов. Определены четыре мультиметода: `append-data-point` (добавление точки в состояние), `has-sufficient-data?` (проверка готовности к интерполяции), `compute-interpolation` (вычисление значения) и `reduce-state-size` (оптимизация памяти). Каждый алгоритм реализует эти методы через `defmethod`.

#### Линейная интерполяция

`locate-interval` - находит пару соседних точек [p1, p2], таких что x1 <= x <= x2. Использует `reduce` с `partition` для создания скользящего окна пар точек и досрочный выход через `reduced`.

`calculate-linear-interpolation` - выполняет линейную интерполяцию по найденному сегменту. Вычисляет параметр t как отношение расстояний, затем интерполирует значение y. Обрабатывает случай совпадения x-координат. Возвращает интерполированное значение или nil.

#### Интерполяция Ньютона

`select-optimal-window` - выбирает окно из n точек, ближайших к значению x. Находит ближайшую к target-x точку, затем берет по n/2 точек слева и справа. Корректирует границы, чтобы окно не выходило за пределы массива.

`build-difference-table` - строит таблицу разделённых разностей рекурсивно через loop/recur. На каждом уровне вычисляет разности соседних элементов, деленные на разность соответствующих x-координат. Возвращает вектор коэффициентов [a0, a1, a2, ...].

`evaluate-newton-polynomial` - вычисляет значение полинома по схеме Горнера. Использует хвостовую рекурсию для эффективного вычисления, начиная с последнего коэффициента.

`calculate-newton-interpolation` - объединяет все этапы: выбор окна, построение таблицы разностей и вычисление значения полинома.

#### Потоковая обработка

`create-initial-state` - создает начальное состояние для всех алгоритмов. Использует PersistentQueue для хранения точек и хранит координату next-x для отслеживания следующей точки вывода.

`generate-outputs-for-algorithm` - генерирует выходные точки для одного алгоритма. Интерполирует с заданным шагом от next-x до upper-bound, накапливая результаты в вектор. Обновляет next-x для следующего вызова.

`generate-all-outputs` - объединяет результаты всех активных алгоритмов, вызывая generate-outputs-for-algorithm для каждого включенного метода.

`handle-new-point` - основная функция потоковой обработки. Добавляет входную точку в состояние каждого активного алгоритма через append-data-point, ограничивает размер состояния через reduce-state-size (удаляет старые точки из очереди), генерирует интерполированные значения до координаты новой точки и возвращает обновленное состояние с массивом выходных точек.

### Модуль `core.clj`

#### Парсинг командной строки

`process-arguments` - парсит аргументы командной строки через loop/recur. Использует cond для обработки различных опций. Валидирует конфигурацию перед возвратом: проверяет наличие хотя бы одного алгоритма и положительность шага. Поддерживает параметры: --linear (линейная интерполяция), --newton (интерполяция Ньютона), --step (шаг дискретизации), -n/--n (количество точек для Ньютона).

#### Обработка входных данных

`extract-point-from-line` - парсит входную строку с поддержкой различных разделителей (пробел, точка с запятой, запятая, табуляция). Использует регулярное выражение для разбиения строки. В случае ошибки выводит предупреждение в stderr через binding, но возвращает nil и позволяет программе продолжить работу (гранулярная обработка ошибок). Возвращает структуру {:x ... :y ...} или nil.

`format-number` - форматирует числа для вывода. Округляет до 3 знаков после запятой, использует US locale для точки как разделителя. Удаляет trailing zeros через регулярное выражение.

`print-result` - выводит результат интерполяции в формате "алгоритм: x y".

#### Основной цикл

`process-input-stream` - основной цикл обработки входного потока. Читает строки через read-line до EOF, парсит точки через extract-point-from-line. Для каждой валидной точки вызывает handle-new-point и выводит результаты через doseq. Использует loop/recur для итерации. При достижении EOF возвращает финальное состояние и координату последней точки.

`finalize-interpolation` - выводит оставшиеся интерполированные точки после EOF. Для каждого активного алгоритма генерирует дополнительные точки с заданным шагом. Реализует требование ТЗ о том, что "последнее окно используется для расчёта большого количества точек".

`-main` - точка входа программы. Последовательно выполняет: парсинг аргументов командной строки, инициализацию состояния интерполятора, чтение и обработку входного потока, финализацию с выводом оставшихся точек.

## Потоковый режим

Программа работает в потоковом режиме: обрабатывает данные по мере поступления. Используется скользящее окно точек фиксированного размера:
- Для линейной интерполяции — 2 точки
- Для интерполяции Ньютона — `n` точек (задается параметром `-n`)

Старые точки удаляются из окна после обработки, что позволяет работать с неограниченными потоками данных.

Концепция скользящего окна:
```
Входные данные:  0  1  2  3  4  5  ...
Window (n=3):   [0][1][2]           <- Интерполируем около 1
                   [1][2][3]        <- Интерполируем около 2
                      [2][3][4]     <- Интерполируем около 3
```

## Функциональное программирование

Реализация следует принципам функционального программирования:

1. **Immutability** — все структуры данных неизменяемы. Используется `PersistentQueue` для скользящего окна.

2. **Чистые функции** — алгоритмы интерполяции не имеют побочных эффектов и всегда возвращают одинаковый результат для одинаковых входных данных.

3. **Разделение ответственности** — ввод/вывод отделён от алгоритмов интерполяции.

4. **Higher-order функции** — используются `map`, `reduce`, `filter` для работы с коллекциями.

5. **Рекурсия** — вместо циклов используется `loop/recur` (хвостовая рекурсия).

6. **Полиморфизм** — мультиметоды обеспечивают единый интерфейс для различных алгоритмов.

## Тесты

Реализовано три типа тестов (всего 9 тестов, 20 assertions, 0 failures):

### Unit-тесты (interpolation_test.clj)

Проверяют корректность отдельных математических функций. Тестируют линейную интерполяцию между двумя и тремя точками, построение таблицы разделенных разностей для квадратичной функции, вычисление полинома Ньютона, выбор оптимального окна точек для интерполяции Ньютона.

### Property-based тесты (properties_test.clj)

Используют библиотеку test.check для проверки математических свойств на случайно сгенерированных данных (200 итераций каждый тест). Проверяют совпадение результатов линейной интерполяции с аналитическим значением (с точностью до 1e-9), а также монотонность интерполяции (результат должен лежать между граничными значениями y).

### Интеграционные тесты (core_test.clj)

Проверяют корректность работы системы в целом. Тестируют парсинг входных строк с различными форматами разделителей (пробел, точка с запятой, запятая), обработку пустых и некорректных строк, потоковую обработку точек с проверкой правильности генерации выходных значений.
